Phase 1 of 5
BIOS Configuration
Power on → press F2 repeatedly → enter BIOS Setup. Make these changes then click APPLY CHANGES at the bottom.
#DoneRecheckActionStatusYour Notes
1.1 Storage → SATA/NVMe Operation → AHCI/NVMe Storage page
Currently set to "RAID On". Select the middle radio button "AHCI/NVMe". This is the single most critical change — without it, Arch cannot see your SSD.
 CRITICAL
1.2 Storage → M.2 PCIe SSD → confirm ON Storage page
Should already be ON. Verify the toggle is blue/enabled. If it's off, your SSD is disabled at hardware level.
 VERIFY
1.3 Connection → UEFI Network Stack → Disabled Connection page
Currently "Enabled" (rightmost). Select "Disabled" (leftmost). This stops the firmware from loading network protocols during boot — the HTTP Boot screen you're seeing is caused by this.
 CHANGE
1.4 Connection → HTTP(s) Boot → OFF Connection page
Toggle the switch to OFF. This eliminates the wireless network selection screen you see on boot.
 CHANGE
1.5 Connection → WLAN + Bluetooth → confirm both checked ✓ Connection page
Both checkboxes should already be checked. You need Wi-Fi for Arch install and Bluetooth for your keyboard/mouse/headset.
 VERIFY
1.6 Update,Recovery → BIOS Recovery from Hard Drive → OFF Update,Recovery page (top section)
Toggle OFF. We're wiping the SSD, so there will be no recovery partition. This Dell feature stores BIOS backup on the disk — useless for our purpose.
 CHANGE
1.7 Update,Recovery → SupportAssist OS Recovery → OFF Update,Recovery page (scroll down)
Toggle OFF. This phones home to Dell's cloud servers to recover Windows — we don't want that.
 CHANGE
1.8 Update,Recovery → BIOSConnect → OFF Update,Recovery page
Toggle OFF. BIOSConnect connects to Dell's cloud for OS/firmware recovery. Phones home. Kill it.
 CHANGE
1.9 Update,Recovery → Dell Auto OS Recovery Threshold → Off Update,Recovery page
Currently set to "2". Select "Off" radio button. Prevents Dell from auto-launching recovery after failed boots (which will happen during install attempts).
 CHANGE
1.10 Update,Recovery → UEFI Capsule Firmware Updates → keep ON Update,Recovery page (top)
Leave this ON! It allows fwupd on Linux to update your BIOS firmware — actually useful for security patches.
 KEEP ON
1.11 Security → Clear TPM → toggle ON, then Apply Security page
The "Clear" toggle under Intel PTT. Turn it ON and click APPLY CHANGES. This wipes any old Windows TPM keys. The toggle resets to OFF after reboot — that's normal. Keep Intel PTT itself ON.
 ONCE
1.12 Security → SMM Security Mitigation → ON Security page (bottom)
Currently OFF. Toggle ON. Protects against System Management Mode firmware attacks. Important for a security-focused build.
 CHANGE
1.13 Pre-boot Behavior → Warnings and Errors → "Continue on Warnings" Pre-boot Behavior page
Currently "Prompt on Warnings and Errors". Change to middle option: "Continue on Warnings". Prevents annoying boot hangs on non-critical warnings while still pausing on real errors.
 CHANGE
1.14 Virtualization → confirm all ON (VT, VT-d, DMA Protection) Virtualization Support page
All three should already be ON from your photos. Verify: Intel VT = ON, VT for Direct I/O = ON, Pre-Boot DMA = ON, OS Kernel DMA = ON. These are needed for Docker, VMs, and security.
 VERIFY
1.15 System Logs → Clear BIOS Event Log → select "Clear Log" then Apply System Logs page
Optional but recommended for a clean start. Select "Clear Log" radio button and apply. Removes old battery warnings and firmware update history.
 OPTIONAL
1.16 Click APPLY CHANGES → counter shows number of changes → EXIT Bottom bar
The orange counter at the bottom should show your total changes (expect 8-10). Click APPLY CHANGES, then click EXIT. System will reboot.
 CRITICAL
Phase 2 of 5
Download Arch ISO & Create Bootable USB
Do this on your current Windows computer (or any working machine) before you reboot the Dell.
#DoneRecheckActionStatusYour Notes
2.1 Download the Arch Linux ISO archlinux.org/download
Scroll to your nearest mirror (Germany: fau.de, rwth-aachen.de). Download the file named archlinux-2025.XX.XX-x86_64.iso (≈900MB). Also download the SHA256 checksum file.
 CRITICAL
2.2 Verify the ISO checksum
Open PowerShell in the Downloads folder and run:
certutil -hashfile archlinux-*.iso SHA256
Compare the output hash with the one on the download page. They must match exactly.
 CRITICAL
2.3 Download Rufus rufus.ie
Download Rufus (portable version is fine, no install needed). It's a ~1MB tool that writes ISOs to USB drives.
 CRITICAL
2.4 Insert USB stick (≥2GB) and open Rufus
⚠️ All data on the USB will be destroyed. Rufus auto-detects the USB drive. Settings: Partition scheme = GPT, Target = UEFI (non CSM), File system = leave default.
 CRITICAL
2.5 Select the Arch ISO in Rufus → click SELECT → choose the .iso file
After selecting, Rufus may ask: "Write in ISO Image mode or DD Image mode?" Select DD Image mode. This preserves the hybrid boot layout the Arch ISO needs.
 DD MODE!
2.6 Click START in Rufus → wait for completion → "READY" appears
Takes 1-5 minutes. When it says READY with a green bar, the USB is bootable. Do NOT remove the USB yet.
 CRITICAL
2.7 Safely eject the USB from Windows
Right-click the USB in File Explorer → Eject. Or use the system tray "Safely Remove Hardware" icon. Wait for the "safe to remove" message.
 SAFE
Phase 3 of 5
Boot from USB into Arch Live Environment
Insert USB into the Dell → power on → get to Arch live shell.
#DoneRecheckActionStatusYour Notes
3.1 Insert USB into Dell → power on → press F12 repeatedly
F12 opens the one-time boot menu (not the BIOS setup — that's F2). You should see your USB listed as "UEFI: [USB name]". Select it and press Enter.
 CRITICAL
3.2 Arch boot menu appears → select first entry "Arch Linux install medium"
Press Enter. Text will scroll for ~10 seconds as the kernel loads. You'll end up at a prompt: root@archiso ~ #
 CRITICAL
3.3 Verify UEFI mode: ls /sys/firmware/efi/efivars
If you see a list of files → you are in UEFI mode ✓. If "No such file or directory" → reboot, enter BIOS (F2), disable CSM/Legacy, try again.
 VERIFY
3.4 Set keyboard: loadkeys de-latin1
Switches to German keyboard layout. Then set a readable font: setfont ter-128b
 DO
3.5 Connect to Wi-Fi: iwctl
Inside iwctl type: station wlan0 connect "FRITZ!Box Fon WLAN 7360" → enter your Wi-Fi password → type exit → then ping -c 3 archlinux.org to confirm.
 CRITICAL
3.6 Sync clock: timedatectl set-ntp true
Ensures accurate time for package signing verification. Verify with timedatectl status — should show "System clock synchronized: yes".
 DO
Phase 4 of 5
Secure Disk Wipe
You said you already cleaned the SSD but aren't 100% sure. We verify and do a final clean pass.
⚠️ POINT OF NO RETURN

After this phase, everything on the SSD is permanently gone. Make absolutely sure you have no data you need on this laptop.

#DoneRecheckActionStatusYour Notes
4.1 Identify your SSD: lsblk
You should see nvme0n1 (your NVMe SSD) and sda (your USB stick). Note the SSD size — it should match your drive (likely 256GB or 512GB). NEVER run wipe commands on the USB device!
 IDENTIFY
4.2 NVMe Secure Erase: nvme format /dev/nvme0n1 --ses=1
This is the fastest and most thorough method. The SSD controller erases all NAND blocks internally, including over-provisioned areas that dd can't touch. Takes seconds. If it says "Format NVM not supported", fall back to: blkdiscard --force /dev/nvme0n1
 WIPE
4.3 Verify wipe: dd if=/dev/nvme0n1 bs=1M count=1 | hexdump -C | head -5
Should show all zeros: 00000000 00 00 00 00... followed by *. Also run fdisk -l /dev/nvme0n1 — should say "no partition table found".
 VERIFY
