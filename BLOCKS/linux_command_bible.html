1. Navigation & File System

ls / eza — List Directory Contents

The ls command lists files and directories. On your Arch system, you will use eza instead — a modern Rust-based replacement with colors, icons, Git integration, and tree view. Think of ls as peeking through a window to see what is inside a room (directory). Each file has metadata: size, owner, permissions, and timestamps.

bash
# Basic listing
ls                    # list files in current directory
ls -la               # -l = long format, -a = show hidden files (starting with .)
ls -lah              # -h = human-readable sizes (KB, MB, GB instead of bytes)
ls -lt               # sort by modification time (newest first)
ls -lS               # sort by size (largest first)
ls -R                # recursive — list all subdirectories too

# Modern replacement — eza (install: pacman -S eza)
eza                   # colorized ls
eza -la              # long + hidden, with git status column
eza --tree -L 3     # tree view, 3 levels deep
eza --icons -la     # show file type icons (needs Nerd Font)
Expected Output (eza -la)
drwxr-xr-x  - mari 15 Mar 14:23 .config
drwxr-xr-x  - mari 15 Mar 10:45 .local
.rw-r--r-- 42 mari 15 Mar 14:20 .bashrc
.rw-r--r--  0 mari 15 Mar 14:15 .zshrc
drwxr-xr-x  - mari 15 Mar 13:00 projects
🔍 Reading the Output — Permission String

The string drwxr-xr-x decodes as: d = directory (- for file, l for symlink), then three groups of three characters: rwx (owner: read+write+execute), r-x (group: read+execute), r-x (others: read+execute). In binary, rwx = 111 = 7, r-x = 101 = 5, so this is permission 755. The kernel checks these bits against the effective UID/GID of the calling process during every open(), stat(), or exec() syscall.

cd / z — Change Directory

bash
cd /home/mari/projects   # absolute path (starts from root /)
cd projects              # relative path (from current directory)
cd ..                     # go up one level (parent directory)
cd ../..                  # go up two levels
cd ~                      # go to home directory (/home/mari)
cd -                      # go to PREVIOUS directory (toggle back and forth)

# Modern replacement — zoxide (install: pacman -S zoxide)
# After using cd for a while, zoxide learns your habits:
z proj                    # jumps to /home/mari/projects (fuzzy match!)
z conf                    # jumps to ~/.config (most-visited match for "conf")
zi                        # interactive selection with fzf

When you type cd /home/mari, the shell calls the chdir() system call, which tells the kernel to change the process's current working directory. The kernel stores the CWD as an inode reference in the process's task_struct. All subsequent relative path lookups start from this inode. The .. entry is a real directory entry stored in every directory's data blocks — it is a hard link pointing to the parent directory's inode. The . entry points to the directory's own inode.

pwd — Print Working Directory

bash
pwd                       # shows your current location in the filesystem
Output
/home/mari/projects/archguide

mkdir / rmdir — Create & Remove Directories

bash
mkdir myproject                    # create a single directory
mkdir -p src/components/ui         # -p creates parent dirs too (entire path)
mkdir -p {src,tests,docs,config}  # brace expansion — creates 4 dirs at once!
rmdir emptydir                     # remove ONLY if empty (safe)

cp — Copy Files and Directories

bash
cp file.txt backup.txt            # copy file
cp -r myproject/ myproject_bak/   # -r = recursive (copy entire directory tree)
cp -i file.txt dest/              # -i = interactive (ask before overwriting)
cp -a source/ dest/               # -a = archive (preserves permissions, timestamps, symlinks)
cp *.py ../scripts/               # glob: copy all .py files to another dir

mv — Move / Rename Files

bash
mv old_name.txt new_name.txt      # rename a file
mv file.txt /home/mari/docs/       # move to another directory
mv -i *.py scripts/               # move all .py files (interactive, asks on conflict)

Under the hood, mv within the same filesystem is extremely fast because it only updates the directory entry (renames the filename→inode mapping). No data is copied. Across filesystems, mv falls back to copy + delete, which is slow for large files. The kernel implements this via the rename() syscall for same-filesystem moves and read()/write()/unlink() for cross-filesystem moves.

rm — Remove Files (⚠️ DANGER ZONE)

⚠️ WARNING — rm IS PERMANENT

Linux has no trash bin by default. rm immediately unlinks the file from the directory and, once no processes hold the file open, the inode and data blocks are freed. There is no "undo" button. Always double-check before using rm -rf.

bash
rm file.txt                        # delete a file
rm -i file.txt                    # -i = ask for confirmation (RECOMMENDED for beginners)
rm -r directory/                   # -r = recursive (delete directory and everything in it)
rm -rf directory/                  # -rf = force + recursive (NO confirmation, NO mercy)

# SAFER ALTERNATIVE: use trash-cli (install: pacman -S trash-cli)
trash-put file.txt                 # moves to trash instead of deleting
trash-list                          # see what's in trash
trash-restore                       # restore from trash

ln — Create Links

bash
# Symbolic link (shortcut — most common, works across filesystems)
ln -s /path/to/original /path/to/link

# Example: link your dotfiles from a git repo to their expected locations
ln -sf ~/dotfiles/.zshrc ~/.zshrc
ln -sf ~/dotfiles/hypr/ ~/.config/hypr

# Hard link (same inode, same data — file exists in two places)
ln original.txt hardlink.txt

A symbolic link is a small file whose content is the path to the target. When you access the symlink, the kernel's VFS layer reads the path stored in the symlink's inode and redirects the operation to the target. A hard link creates a second directory entry pointing to the same inode — the file effectively has two names. The inode's link count tracks how many directory entries reference it; the file is only deleted when the link count drops to zero AND no processes have it open.

find / fd — Search for Files

bash
# Traditional find
find . -name "*.py"                # find all .py files recursively from current dir
find /home -type f -size +100M   # files larger than 100MB
find . -name "*.log" -mtime +7 -delete  # delete .log files older than 7 days

# Modern replacement — fd (install: pacman -S fd)
fd "\.py$"                          # find .py files (regex by default, respects .gitignore)
fd -e py                            # same but by extension
fd -H config                       # -H includes hidden files
fd -t d                             # only directories
fd -x wc -l "\.py$"              # find .py files and count lines in each

2. File Viewing & Text Processing

cat / bat — Display File Contents

bash
cat file.txt                       # dump entire file to terminal
cat file1.txt file2.txt            # concatenate (that's what cat stands for!)
cat file1.txt file2.txt > merged.txt  # merge into new file

# Modern replacement — bat (install: pacman -S bat)
bat main.py                        # syntax highlighting + line numbers + git diff
bat -l json data.txt              # force JSON syntax highlighting
bat --diff file.py                # show only git-modified lines

less / more — Page Through Files

bash
less largefile.log                 # open file in pager — scroll with arrows, q to quit
# Inside less:
#   /search_term  → search forward
#   ?search_term  → search backward
#   n/N           → next/previous match
#   g/G           → go to beginning/end
#   q             → quit

# Pipe anything into less for scrollable output
journalctl | less
dmesg | less

head / tail — View Beginning or End

bash
head -20 file.txt                 # first 20 lines
tail -20 file.txt                 # last 20 lines
tail -f /var/log/syslog            # -f = FOLLOW (watch log in real-time — Ctrl+C to stop)
tail -f app.log | grep ERROR     # live-filter for ERROR lines only

grep / ripgrep — Search Text

bash
# Traditional grep
grep "error" logfile.txt           # find lines containing "error"
grep -i "error" logfile.txt       # -i = case-insensitive
grep -r "TODO" src/               # -r = recursive search through directories
grep -n "def " *.py               # -n = show line numbers
grep -c "import" main.py         # -c = count matching lines
grep -v "#" config.conf           # -v = INVERT (show lines NOT matching — skip comments)

# Modern replacement — ripgrep (install: pacman -S ripgrep)
rg "TODO"                           # recursive search (automatic! respects .gitignore)
rg -i "error" -t py              # case-insensitive, only .py files
rg "def \w+" --stats              # regex + show match statistics
rg -l "import pandas"             # -l = list only filenames (not matching lines)

sed — Stream Editor

sed processes text line by line and applies transformations. It is the Swiss Army knife of text manipulation in Linux. Think of it as "Find and Replace" for the command line, but with regex superpowers.

bash
# Find and replace (first occurrence per line)
sed 's/old/new/' file.txt

# Find and replace ALL occurrences per line (g = global)
sed 's/old/new/g' file.txt

# Edit file IN-PLACE (overwrites the original — careful!)
sed -i 's/old/new/g' file.txt

# Delete lines matching a pattern
sed '/^#/d' config.conf           # remove all comment lines
sed '/^$/d' file.txt               # remove all blank lines

# Print specific lines
sed -n '10,20p' file.txt          # print lines 10 through 20 only

# Practical example: uncomment a config line
sed -i 's/^#PermitRootLogin/PermitRootLogin/' /etc/ssh/sshd_config

awk — Text Processing Language

bash
# Print specific columns (awk splits by whitespace by default)
awk '{print $1}' file.txt          # first column of each line
awk '{print $1, $3}' file.txt     # first and third columns

# Use custom delimiter
awk -F: '{print $1}' /etc/passwd   # : as delimiter — prints all usernames

# Filter rows with conditions
awk '$3 > 1000' data.txt          # rows where 3rd column is > 1000

# Practical: sum a column of numbers
awk '{sum += $2} END {print sum}' data.txt

# Show disk usage by mount point, sorted
df -h | awk 'NR>1 {print $5, $6}' | sort -rn

sort / uniq / wc — Sort, Deduplicate, Count

bash
sort file.txt                      # alphabetical sort
sort -n numbers.txt               # numeric sort
sort -r file.txt                  # reverse sort
sort -u file.txt                  # unique lines only (sort + dedup in one step)

uniq sorted.txt                   # remove consecutive duplicates (MUST be sorted first!)
sort file.txt | uniq -c          # count occurrences of each unique line

wc file.txt                        # lines, words, bytes
wc -l file.txt                    # count lines only
wc -l *.py                        # count lines in all Python files

3. Permissions & Ownership

chmod — Change Permissions

bash
# Numeric mode (most common)
chmod 755 script.sh   # rwxr-xr-x  owner=all, group/others=read+execute
chmod 644 file.txt    # rw-r--r--  owner=read+write, group/others=read only
chmod 600 id_rsa      # rw-------  owner only (SSH private key MUST be this)
chmod 700 ~/.ssh      # rwx------  SSH directory

# Symbolic mode (add/remove specific permissions)
chmod +x script.sh   # add execute permission for everyone
chmod u+x script.sh  # add execute for owner (u=user) only
chmod go-w file.txt  # remove write from group and others
chmod -R 755 dir/    # recursive — apply to all files in directory
🔍 Permission Numbers Decoded

Each digit is a sum: 4=read, 2=write, 1=execute. So 7=4+2+1=rwx, 5=4+1=r-x, 6=4+2=rw-, 0=---. The three digits are: owner, group, others. For scripts you want to run: 755. For private config files: 600. For SSH keys: 600 (SSH refuses to use keys with broader permissions).

chown — Change Ownership

bash
sudo chown mari file.txt           # change owner to mari
sudo chown mari:wheel file.txt    # change owner AND group
sudo chown -R mari:mari ~/proj/  # recursive — entire directory tree

sudo — Execute as Root

bash
sudo pacman -Syu                   # run pacman as root
sudo -i                            # open a root shell (type 'exit' to return)
sudo -u postgres psql             # run command as the postgres user
sudo !!                            # re-run the LAST command with sudo (lifesaver!)

4. Process Management

ps / procs — List Processes

bash
ps aux                             # all processes, all users, with details
ps aux | grep python              # find Python processes
htop                               # interactive process viewer (F9=kill, F6=sort)
btop                               # beautiful process viewer with CPU/RAM/disk/network graphs

kill / killall — Terminate Processes

bash
kill 1234                           # send SIGTERM (polite shutdown) to PID 1234
kill -9 1234                       # send SIGKILL (force kill — cannot be caught or ignored)
killall firefox                    # kill all processes named firefox
pkill -f "python main.py"         # kill by command line pattern

bg / fg / jobs / & — Background & Foreground

bash
command &                          # start command in background
# Press Ctrl+Z to SUSPEND a running foreground process
bg                                 # resume suspended process in BACKGROUND
fg                                 # bring background process to FOREGROUND
jobs                               # list all background jobs
nohup command &                   # run command immune to hangups (survives terminal close)

systemctl — Manage Services

bash
systemctl status NetworkManager    # check if a service is running
systemctl start sshd              # start a service NOW
systemctl stop sshd               # stop a service NOW
systemctl restart sshd            # restart (stop + start)
systemctl enable sshd             # start automatically on every boot
systemctl disable sshd            # DON'T start on boot
systemctl enable --now docker     # enable AND start immediately (one command)
systemctl list-units --type=service  # list all running services
journalctl -u sshd -f             # live log for a specific service

5. Networking

ip — Network Configuration

bash
ip addr                            # show all network interfaces and their IPs
ip link                            # show interface status (UP/DOWN)
ip route                           # show routing table (default gateway)
ip addr show wlan0                # show specific interface details

ss — Socket Statistics

bash
ss -tulnp                          # show all listening TCP/UDP ports with process names
# -t=TCP -u=UDP -l=listening -n=numeric -p=show process
ss -tunp | grep :8080             # who's using port 8080?

ping / traceroute / dig — Diagnostics

bash
ping -c 4 archlinux.org            # send 4 ICMP packets
traceroute archlinux.org           # show network path (every router hop)
dig archlinux.org                  # DNS lookup (what IP does this hostname resolve to?)
dig +short archlinux.org           # just the IP, nothing else

curl — HTTP Swiss Army Knife

bash
curl https://api.example.com       # GET request
curl -o file.zip https://url/file.zip  # download file
curl -X POST -H "Content-Type: application/json" \
  -d '{"key":"value"}' https://api.example.com  # POST JSON
curl -I https://example.com        # headers only (check HTTP status)
curl -sS https://api.ip.sb/ip     # get your public IP address

ssh / scp — Secure Remote Access

bash
ssh user@hostname                  # connect to remote machine
ssh -p 2222 user@host             # custom port
ssh-keygen -t ed25519              # generate SSH key pair (ed25519 is modern & fast)
ssh-copy-id user@host              # copy your public key to remote (enables passwordless login)
scp file.txt user@host:~/         # copy file to remote home directory
scp -r dir/ user@host:~/          # copy directory recursively

6. Package Management (pacman + paru)

pacman — The Arch Package Manager

bash
# ═══ INSTALLING ═══
sudo pacman -S package             # install a package
sudo pacman -S pkg1 pkg2 pkg3     # install multiple at once

# ═══ UPDATING ═══
sudo pacman -Syu                   # sync repos + upgrade ALL packages (do this regularly!)
sudo pacman -Sy                    # sync repos only (refresh package database)

# ═══ REMOVING ═══
sudo pacman -R package             # remove package
sudo pacman -Rs package            # remove + its unused dependencies
sudo pacman -Rns package           # remove + deps + config files (complete removal)

# ═══ SEARCHING ═══
pacman -Ss keyword                # search repos for packages matching keyword
pacman -Qs keyword                # search INSTALLED packages
pacman -Qi package                # detailed info about an installed package
pacman -Ql package                # list all files installed by a package
pacman -Qo /path/to/file         # which package owns this file?

# ═══ MAINTENANCE ═══
sudo paccache -r                  # remove old cached packages (keep last 3 versions)
pacman -Qdt                       # list orphaned packages (installed as deps, no longer needed)
sudo pacman -Rns $(pacman -Qdtq)  # remove all orphans

paru — AUR Helper

bash
paru -S package                    # install from AUR (or official repos)
paru                               # same as paru -Syu (update everything including AUR)
paru -Ss keyword                  # search AUR + official repos
paru -c                            # clean unneeded dependencies

7. Disk & Storage

lsblk / fdisk / df / du — Disk Info

bash
lsblk                              # list all block devices (disks, partitions, USBs)
lsblk -f                           # show filesystem type, UUID, mount point
df -h                              # disk space usage per mount point (-h = human readable)
du -sh directory/                 # total size of a directory

dust                               # modern du (install: pacman -S dust) — visual bars
sudo fdisk -l                     # list partition tables on all disks
sudo smartctl -a /dev/nvme0n1    # SSD health (SMART data)

mount / umount — Mount Filesystems

bash
sudo mount /dev/sdb1 /mnt/usb    # mount a USB drive
sudo umount /mnt/usb              # unmount
mount | grep "nvme"               # show all mounted NVMe partitions

9. Git Essentials

bash

10. Docker Quick Reference

bash
# ═══ SETUP (one-time) ═══
sudo systemctl enable --now docker
sudo usermod -aG docker mari      # add yourself to docker group (logout/login after)

# ═══ IMAGES ═══
docker pull python:3.12            # download image
docker images                      # list local images
docker rmi image_name              # remove image

# ═══ CONTAINERS ═══

docker run -d -p 8080:80 nginx    # run in background, map port 8080→80
docker ps                           # list running containers
docker ps -a                       # list ALL containers (including stopped)
docker stop container_id           # stop a container
docker logs -f container_id       # follow container logs
docker exec -it container_id bash  # shell into running container

# ═══ COMPOSE ═══
docker compose up -d               # start all services defined in docker-compose.yml
docker compose down                # stop and remove all services
docker compose logs -f             # follow all logs
